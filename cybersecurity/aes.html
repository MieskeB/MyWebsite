AES

Calculating AES by hand

Blocks

For this AES encryption I am going to encrypt the following plaintext (p) sentence: "THIS IS A COOL MESSAGE". I am going to use the key (k): "ABCDEFGHIJKLMNOP" (which has a length of 16 characters).

In AES, everything is divided in blocks of 128 bits (1 byte per square = 8 bits; a block is 4x4 so 16 squares; 16 * 8 = 128 bits). This means that the first part of the plaintext will look like as follows:

T A O
H I L
I S C
S O M

The key will look as follows:

A E I M
B F J N
C G K O
D H L P

Next up, we need to convert both blocks to hexadecimals. First the plaintext:
54 20 41 4f
48 49 20 4c
49 53 43 20
53 20 4f 4d

Then the key converted to hexadecimals:
41 45 49 4d
42 46 4a 4e
43 47 4b 4f
44 48 4c 50

Adding the keys (xor)

The first step is adding the keys to the plaintext. This is done with a binary xor. So first 54 (1010100) ^ 41 (1000001) means that 1 and 1 becomes 0, 1 and 0 becomes 1, 0 and 0 becomes 0. So this number will be 15 (0010101). Doing this with the whole block, will result in the following block.
15 65 08 02
0a 0f 6a 02
0a 14 08 6f
17 68 03 1d

Shift

The next thing that needs to be done is shifting the rows. The 1st row won't shift, the 2nd row will shift 1 to the left, the 3rd row 2 and the 4th row 3. This will give the following data:
2f bc bf 6a
fb 58 6a 47
ff 06 47 16
de 87 ea bb

Mix

This step is very difficult to do by hand and thus I will skip it. But in a nutshell what I think this step is: Every square is bitshifted x times to the left.

Repeat

After having done everything, this process is repeated from the xor part. It is repeated ten times for an 128 bit key. After that, the string is encrypted via AES.
